## Design Language (v3 - Premium SaaS Polish)
- **Color palette**: Slate-based (not gray) for all neutral colors across the entire app
- **Primary gradient**: Blue-to-cyan (`from-blue-600 to-cyan-600`)
- **Primary button**: `btn-primary` class (gradient blue, rounded-xl, shadow, active:scale)
- **Secondary button**: `btn-secondary` class (white bg, slate border, rounded-xl)
- **Brand gradient text**: `.text-gradient` class (blue→cyan→purple)
- **Card style**: `bg-white rounded-2xl shadow-sm border border-slate-100/60` with `.card-hover` and `.premium-card` for interactive cards
- **Input style**: `premium-input` class (rounded-xl, border-slate-200, hover:border-slate-300, focus:ring-blue-500/20)
- **Select style**: `premium-select` class (rounded-xl, custom chevron, same focus as inputs)
- **Table style**: `premium-table` class (uppercase tracking-wider headers, bg-slate-50/80, hover:bg-blue-50/30 rows)
- **Metric cards**: `.metric-icon` with color variants (`.metric-icon-blue`, `.metric-icon-green`, `.metric-icon-violet`, `.metric-icon-amber`, `.metric-icon-rose`)
- **Status badges**: `.badge-success`, `.badge-warning`, `.badge-danger`, `.badge-info`, `.badge-neutral` CSS classes
- **Empty states**: `.empty-state`, `.empty-state-icon`, `.empty-state-title`, `.empty-state-description` CSS classes
- **Skeleton loaders**: `.skeleton-shimmer` class (shimmer animation, replaces old spinners)
- **Animations**: `animate-fade-in-up`, `animate-scale-in`, `animate-slide-down`, `animate-count-up`, `animate-float` + `.stagger-1` through `.stagger-8`
- **Glass effects**: `.glass` and `.glass-dark` classes (backdrop-blur + saturate)
- **Modal overlays**: `bg-slate-900/50 backdrop-blur-sm` (not bg-black/50)
- **Sidebar**: 248px width, bg-white, border-slate-200/40, sectioned nav with gradient active states
- **TopBar**: bg-white/90 backdrop-blur-xl, subtle shadow, rounded-xl elements
- **Login page**: Dark left panel with animated grid, glowing orbs, feature cards; light right panel with clean form
- **Main layout background**: `bg-gradient-to-br from-slate-50 via-white to-blue-50/30`
- **Dashboard**: Skeleton loading states, hero KPI card (blue gradient), premium chart tabs
- **CSS variables**: --primary updated to blue (221.2 83.2% 53.3%), --radius to 0.75rem
- **Typography**: antialiased, tracking-tight on headings, font-feature-settings for OpenType
- **DB optimization**: Dashboard API queries split into 2 batches of 10 to avoid connection limit

## Layout Architecture
- **TopBar** (`components/top-bar.tsx`): Sticky header bar with company badge (left), search bar (center), help/notifications/settings/user profile (right)
- **Sidebar** (`components/sidebar.tsx`): Navigation-only, receives `mobileOpen` and `onCloseMobile` props, `sticky top-14`, width 240px, collapsible
- **MainLayout** (`components/main-layout.tsx`): Manages mobile sidebar state, renders TopBar → flex(Sidebar + main)
- **NotificationBell**: Accepts `position` prop ('sidebar' | 'header'). Header mode opens dropdown downward-right.
- **Mobile**: Toggle button in TopBar, sidebar overlay with backdrop

## Technical Patterns
- **documentCode handling**: ARCA codes stored as zero-padded strings ('003'). Always use `parseInt()` for comparisons.
- **NC/ND identification**: documentCode '003'/'008' (NC), '002'/'007' (ND) — use parseInt for matching
- **DB shared**: Dev and prod share the same database — be careful with data operations
